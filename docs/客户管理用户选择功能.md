# 客户管理 - 用户选择功能

## 📋 功能说明

客户管理新增客户时，必须从用户管理中选择已有用户，不能自行录入。

### ✅ 核心功能
1. **用户选择器** - 点击"选择用户"按钮打开选择器弹窗
2. **搜索功能** - 支持按手机号、姓名、ID 查询用户
3. **自动填充** - 选择后自动填充客户名称和手机号
4. **后端验证** - API 验证 user_id 必须存在于 users 表
5. **状态检查** - 只能选择状态为正常的用户（status=1）

## 📊 数据库变更

### customers 表新增字段

```sql

```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| user_id | INT | 0 | 关联 users 表的用户 ID |

## 🎯 API 变更

### api/user.php - 新增搜索接口

**请求：**
```
GET /api/user.php?action=search&keyword=张三
```

**响应：**
```json
{
    "code": 200,
    "data": [
        {
            "id": 1,
            "username": "张三",
            "nickname": "张三",
            "phone": "13800138000",
            "email": "zhangsan@example.com",
            "role": 1,
            "status": 1
        }
    ]
}
```

**搜索规则：**
- 如果 keyword 是数字，支持 ID 精确匹配 + 手机号/姓名模糊匹配
- 如果 keyword 是文字，支持姓名/昵称/手机号模糊匹配
- 只返回 status=1 的正常用户
- 最多返回 20 条结果

### api/customer.php - 创建/更新验证

**创建客户 (create)：**
1. 验证 name 不为空
2. 验证 user_id 不为空
3. 验证 user_id 在 users 表中存在且 status=1

**更新客户 (update)：**
1. 验证 name 不为空
2. 验证 user_id 不为空
3. 验证 user_id 在 users 表中存在且 status=1

**错误响应：**
```json
{
    "code": 400,
    "message": "必须从用户管理中选择已有用户"
}
```

或

```json
{
    "code": 400,
    "message": "选择的用户不存在或已禁用"
}
```

## 💡 使用流程

### 新增客户

1. 点击"➕ 新增客户"按钮
2. 点击"🔍 选择用户"按钮
3. 在弹窗中输入搜索条件（手机号/姓名/ID）
4. 点击"🔍 搜索"按钮
5. 在搜索结果中点击"✅ 选择"按钮
6. 自动填充客户名称和手机号
7. 填写其他信息（国家、岗位、来源、状态等）
8. 点击"保存"按钮

### 编辑客户

1. 点击客户列表中的"📝 编辑"按钮
2. 客户名称字段已填充（只读）
3. 如需更换用户，点击"🔍 选择用户"重新选择
4. 修改其他信息
5. 点击"保存"按钮

## ⚠️ 注意事项

1. **必填验证** - 客户名称和用户 ID 都是必填
2. **只读字段** - 客户名称输入框为只读，必须通过选择器选择
3. **自动填充** - 选择用户后自动填充手机号
4. **用户状态** - 只能选择 status=1 的正常用户
5. **后端验证** - 前端验证 + 后端验证双重保障
