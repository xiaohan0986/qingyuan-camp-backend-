# 数据大屏卡片拖拽排序

## 📋 功能说明

数据大屏的统计卡片支持拖拽排序，布局自动保存到数据库。

### ✅ 核心功能
1. **直接拖拽** - 无需点击编辑按钮，直接拖动卡片
2. **自动保存** - 拖拽结束后自动保存到数据库
3. **数据库存储** - 布局保存在 `dashboard_layout` 表
4. **用户独立** - 每个用户有独立的布局配置

## 📊 数据库表结构

```sql
CREATE TABLE `dashboard_layout` (
    `id` INT AUTO_INCREMENT PRIMARY KEY,
    `user_id` INT NOT NULL COMMENT '用户 ID',
    `first_row` TEXT NOT NULL COMMENT '第一排卡片顺序 (JSON)',
    `second_row` TEXT NOT NULL COMMENT '第二排卡片顺序 (JSON)',
    `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY `unique_user` (`user_id`),
    KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| user_id | INT | 用户 ID（唯一） |
| first_row | TEXT | 第一排卡片顺序（JSON 数组） |
| second_row | TEXT | 第二排卡片顺序（JSON 数组） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### JSON 数据格式

```json
// first_row 示例
[
    "👁️ 总访问数",
    "👤 总用户数",
    "👥 总客户数",
    "💼 总岗位数",
    "💰 已成交客户"
]

// second_row 示例
[
    "📰 文章总数",
    "➕ 今日新增文章",
    "📖 文章总阅读数",
    "👤 今日新增用户"
]
```

## 🎯 API 接口

### 获取布局
```
GET /api/dashboard.php?action=get_layout
```

响应：
```json
{
    "code": 200,
    "data": {
        "first_row": ["👁️ 总访问数", "👤 总用户数"],
        "second_row": ["📰 文章总数", "➕ 今日新增文章"]
    }
}
```

### 保存布局
```
POST /api/dashboard.php?action=save_layout
Content-Type: application/json

{
    "first_row": ["👁️ 总访问数", "👤 总用户数"],
    "second_row": ["📰 文章总数", "➕ 今日新增文章"]
}
```

响应：
```json
{
    "code": 200,
    "data": {
        "message": "布局已保存"
    }
}
```

## 💡 使用提示

1. **直接拖动** - 鼠标按住卡片即可拖动
2. **自动保存** - 松开鼠标后自动保存，显示"✅ 位置已保存"提示
3. **跨设备同步** - 布局保存在数据库，不同设备登录同一账号布局一致
4. **重置布局** - 暂时不支持，需要手动调整或清空数据库记录

## 🔧 技术实现

### 前端
- **HTML5 Drag & Drop API** - 原生拖拽功能
- **MutationObserver** - 监听 DOM 变化
- **Fetch API** - 异步保存布局

### 后端
- **Session 认证** - 基于用户 ID 保存布局
- **JSON 存储** - 卡片顺序保存为 JSON 格式
- **自动更新** - 使用 `ON UPDATE CURRENT_TIMESTAMP`
