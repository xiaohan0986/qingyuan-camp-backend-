# 客户管理 - 国家与岗位联动筛选

## 📋 功能说明

新增客户时，选择国家后岗位下拉框自动筛选出对应国家的岗位。如果未选择国家，选择岗位时会提醒先选择国家。

### ✅ 核心功能
1. **国家选择联动** - 选择国家后自动加载对应国家的岗位
2. **岗位筛选** - 岗位下拉框只显示所选国家的岗位
3. **未选国提示** - 未选择国家时尝试选岗位会弹出警告
4. **默认提示** - 未选国家时岗位框显示"请先选择国家"
5. **空数据处理** - 如果国家没有岗位，显示"该国暂无岗位"

## 🎯 API 变更

### api/customer.php - getPositions 接口

**请求（全部岗位）：**
```
GET /api/customer.php?action=positions
```

**请求（按国家筛选）：**
```
GET /api/customer.php?action=positions&country=日本
```

**响应：**
```json
{
    "code": 200,
    "message": "success",
    "data": [
        {
            "id": 1,
            "title": "软件工程师"
        },
        {
            "id": 2,
            "title": "产品经理"
        }
    ]
}
```

**筛选逻辑：**
- 如果传入 country 参数，只返回该国家的岗位
- 如果未传入 country 参数，返回全部岗位
- 只返回 status=1 的正常岗位

## 💡 使用流程

### 新增客户

1. 点击"➕ 新增客户"按钮
2. 选择用户（通过用户选择器）
3. **选择国家** - 从下拉框选择国家（如"日本"）
4. **岗位自动筛选** - 岗位下拉框自动更新为日本的岗位
5. 选择岗位 - 从筛选后的岗位中选择
6. 填写其他信息（来源、状态、销售顾问等）
7. 点击"保存"按钮

### 修改国家

1. 如果已选择岗位，修改国家会清空岗位选择
2. 岗位下拉框更新为新国家的岗位列表
3. 需要重新选择岗位

## ⚠️ 注意事项

### 前端验证
1. **未选国家提示** - 未选择国家时点击岗位下拉框会弹出警告：
   ```
   ⚠️ 请先选择国家，再选择岗位！
   ```
2. **自动重置** - 警告后岗位选择自动重置为"0"

### 后端逻辑
1. **国家筛选** - API 根据 country 参数筛选岗位
2. **空数据处理** - 如果国家没有岗位，返回空数组
3. **状态检查** - 只返回 status=1 的正常岗位

### 数据一致性
1. **岗位国家** - 需要在岗位管理中为每个岗位设置国家
2. **国家匹配** - 客户选择的国家必须与岗位的国家一致

## 🔧 JavaScript 函数

### loadPositionsByCountry()
国家选择时触发，加载对应国家的岗位列表。

**逻辑：**
1. 获取选中的国家
2. 如果国家为空，岗位框显示"请先选择国家"
3. 调用 API 获取该国家的岗位
4. 更新岗位下拉框选项

### checkCountryBeforePosition()
岗位选择时触发，检查是否已选择国家。

**逻辑：**
1. 检查国家是否为空
2. 如果国家为空且岗位不为"0"，弹出警告
3. 自动重置岗位选择为"0"
