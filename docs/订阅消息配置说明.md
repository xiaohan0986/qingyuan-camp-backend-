# 订阅消息推送配置说明

## 📋 功能概述

办理进度订阅消息推送功能，允许用户订阅订单进度通知。当后台更新办理进度时，系统会自动向已订阅的用户发送微信订阅消息。

## 🔧 配置步骤

### 第一步：微信公众平台配置

1. **登录微信公众平台**
   - 访问：https://mp.weixin.qq.com
   - 使用小程序管理员微信扫码登录

2. **添加订阅消息模板**
   - 左侧菜单：**功能** → **订阅消息**
   - 点击 **"选用公共模板"**
   - 搜索关键词：**"进度"** 或 **"订单"**
   - 推荐模板：
     - 订单进度通知
     - 服务进度通知
     - 事务提醒
   - 点击"选用"，填写关键词（如：办理进度、签证进度等）
   - 提交审核（通常很快通过）

3. **获取模板 ID**
   - 审核通过后，在 **"我的模板"** 中找到刚添加的模板
   - 复制 **模板 ID**（格式类似：`wxxxxxxxxxxxxxxxxxxx`）

### 第二步：后台配置

1. **登录管理后台**
   - 访问：http://你的域名/admin/config.php
   - 使用管理员账号登录

2. **配置订阅推送**
   - 找到 **"订阅推送"** 配置区块（在页面底部）
   - 填写配置：
     - **开启订阅推送**：打开开关
     - **模板 ID**：粘贴从微信公众平台复制的模板 ID
     - **模板名称**：填写备注名称（如：办理进度通知）
     - **自动引导订阅**：打开后用户进入进度页面会自动弹出订阅窗口
     - **进度更新时自动发送**：打开后后台更新进度时自动推送消息
   - 点击 **"保存配置"**

3. **测试发送**
   - 点击 **"测试发送"** 按钮
   - 输入测试用户的 OpenID（可在小程序开发工具中获取）
   - 确认发送

### 第三步：数据库配置

1. **执行 SQL 脚本**
   - 打开 phpMyAdmin 或其他数据库管理工具
   - 选择你的数据库
   - 执行 SQL：`D:\phpstudy_pro\WWW\www.gofong.com\database\user_subscriptions.sql`
   - 创建 `user_subscriptions` 表

### 第四步：小程序端测试

1. **打开小程序**
   - 在微信开发者工具中打开小程序项目
   - 进入办理进度页面

2. **订阅消息**
   - 页面顶部会显示 **"订阅进度通知"** 横幅
   - 点击横幅或等待自动弹出订阅窗口
   - 点击 **"允许"** 授权

3. **接收通知**
   - 后台更新订单进度后
   - 用户会在微信中收到订阅消息提醒
   - 点击消息可跳转到办理进度页面

## 🔍 常见问题

### Q1: 用户收不到订阅消息？
**A:** 检查以下几点：
1. 用户是否已授权订阅（每人每次授权只有 1 次推送机会）
2. 订阅配置中的模板 ID 是否正确
3. 小程序 APPID 和 SECRET 是否正确配置
4. 查看后台日志是否有发送失败的记录

### Q2: 如何引导用户多次订阅？
**A:** 在进度页面添加订阅按钮，每次用户点击都会重新请求授权。可以在关键进度节点提示用户再次订阅。

### Q3: 订阅消息的字段如何对应？
**A:** 根据你选择的模板字段来对应：
- `thing1`：进度阶段名称（如：审核中）
- `thing2`：进度内容（如：材料已提交，等待审核）
- `time3`：更新时间（如：2024-01-01 12:00:00）
- `thing4`：备注（如：需补充材料：身份证、户口本）

### Q4: 如何查看订阅消息发送日志？
**A:** 查看 PHP 错误日志文件，通常在：
- Windows: `D:\phpstudy_pro\php\php8.x.x\php_error_log`
- 或在代码中 `error_log()` 输出的位置

## 📊 数据库表结构

### user_subscriptions 表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键 |
| user_id | INT | 用户 ID |
| openid | VARCHAR(64) | 用户微信 openid |
| template_id | VARCHAR(64) | 订阅消息模板 ID |
| status | ENUM | 订阅状态：active/inactive |
| subscribe_count | INT | 订阅次数 |
| last_used_at | TIMESTAMP | 最后一次发送时间 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

## 🎯 使用建议

1. **引导用户订阅**：在用户提交订单后，立即引导用户订阅进度通知
2. **关键节点推送**：只在关键进度节点发送消息，避免频繁打扰用户
3. **提供退订选项**：在消息中说明如何取消订阅
4. **测试充分**：上线前充分测试，确保消息能正常发送
