# 后台主动推送订阅消息说明

## 📋 功能概述

当后台管理员在 `reports2.php` 页面点击"更新进度"后，系统会：
1. 保存进度记录到数据库
2. **自动获取该用户的 OpenID**
3. **向用户发送微信订阅消息通知**

---

- ✅ 用户有有效的 OpenID
- ✅ 后台配置了小程序 APPID 和 SECRET

## 📊 数据流程

```
后台 reports2.php
    ↓
点击"更新进度"
    ↓
前端 JavaScript (addProgress 函数)
    ↓
POST → ../api/customer.php?action=add_progress
    ↓
后端 PHP (addProgress 函数)
    ├── 1. 保存进度到 customer_progress 表
    ├── 2. 查询客户信息（包括 openid）
    ├── 3. 读取小程序配置（APPID、SECRET、模板 ID）
    ├── 4. 构造订阅消息数据
    └── 5. 调用微信 API 发送消息
    ↓
返回结果给前端
```

## ⚠️ 注意事项

### **1. 用户未订阅**

如果用户未订阅，发送会失败（错误码：43101）

**解决方案**：
- 提示用户在小程序中订阅
- 或者在后台配置中开启"自动引导订阅"

### **2. 配置不完整**

如果缺少 APPID、SECRET 或模板 ID，不会发送消息

**解决方案**：
- 访问：`http://你的域名/admin/config.php`
- 在"基础配置"中填写 APPID 和 SECRET
- 在"订阅推送配置"中填写模板 ID

### **3. OpenID 为空**

如果用户没有 OpenID，无法发送消息

**解决方案**：
- 用户需要在小程序中登录授权
- 或者在数据库中手动关联 OpenID
