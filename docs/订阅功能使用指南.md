2. 选择你的数据库

1. **登录微信公众平台**

4. **获取模板 ID**
   - 审核通过后，进入 **"我的模板"**
   - 复制 **模板 ID**（格式：`wxxxxxxxxxxxxxxxxxxx`）

## 📱 小程序端订阅入口

### 订阅按钮位置

用户在小程序端有 **2 个订阅入口**：

#### 入口 1：用户信息区域订阅按钮
- **位置**：办理进度页面顶部，用户头像和手机号下方
- **样式**：紫色渐变按钮，带铃铛图标 🔔
- **文字**："订阅通知"
- **特点**：带有脉冲动画，非常醒目
- **显示条件**：用户未订阅且配置了模板 ID

#### 入口 2：订阅提示横幅
- **位置**：当前进度卡片上方
- **样式**：紫色渐变横幅，带铃铛图标
- **文字**："订阅进度通知 - 进度更新时第一时间通知您"
- **按钮**："立即订阅"
- **显示条件**：用户未订阅且配置了模板 ID

### 自动订阅功能

如果后台开启了 **"自动引导订阅"**：
- 用户首次进入办理进度页面
- 延迟 1 秒后自动弹出订阅授权窗口
- 用户点击"允许"即可完成订阅

## 🎯 使用流程

### 用户端操作

1. **打开小程序**
   - 进入"办理进度"页面

2. **点击订阅按钮**
   - 点击用户信息下方的"订阅通知"按钮
   - 或点击页面上方的订阅横幅

3. **授权订阅**
   - 系统弹出订阅授权窗口
   - 显示订阅消息模板内容
   - 用户点击"允许"

4. **接收通知**
   - 后台更新订单进度时
   - 用户微信会收到订阅消息提醒
   - 点击消息可跳转到办理进度页面

### 后台操作

1. **更新订单进度**
   - 方法 1：在后台订单管理页面更新进度
   - 方法 2：调用 API `api/update_progress.php`

2. **自动推送**
   - 系统自动向已订阅用户发送消息
   - 无需手动操作

## ❓ 常见问题

### Q1: 订阅按钮不显示？
**A:** 检查以下几点：
1. 后台是否配置了模板 ID
2. 订阅推送开关是否打开
3. 用户是否已经订阅过（已订阅则不显示）
4. 小程序是否重新编译加载

### Q2: 用户收不到推送消息？
**A:** 检查：
1. 用户是否已授权订阅（每人每次授权只有 1 次推送机会）
2. 小程序 APPID 和 SECRET 是否正确
3. 模板 ID 是否正确
4. 查看 PHP 错误日志

### Q3: 如何让用户多次订阅？
**A:** 
- 每次用户点击订阅按钮都会重新请求授权
- 可在关键进度节点提示用户再次订阅
- 或在页面添加"再次订阅"按钮

### Q4: 数据库表创建失败？
**A:** 
- 检查数据库连接信息
- 检查用户权限是否有 CREATE TABLE 权限
- 手动执行 SQL 文件创建表
